# موزّع رأس المال (Capital Allocator)

الأداة الأولى ضمن مجموعة **Mr Amwal Tools** المفتوحة المصدر.

أداة تعليمية بسيطة تساعد المستثمرين المبتدئين والمتوسطين على تقسيم رأس مالهم
قبل الدخول إلى السوق، بدلاً من استثمار كل المبلغ دفعة واحدة بدافع عاطفي.
يُدخل المستخدم إجمالي رأس ماله، وتقترح الأداة توزيعه على أربع فئات:

- **الدخول الأول للسوق** — الجزء المستخدم لأول عملية شراء
- **فرص الشراء المستقبلية** — احتياطي لاستغلال فرص لاحقة
- **السيولة المتاحة** — مبلغ نقدي جاهز للاستخدام المرن
- **الاحتياطي الطارئ** — مبلغ خارج السوق للظروف الطارئة

> **هذه الأداة لا تتنبأ بالسوق، ولا تُصدر توصيات أو إشارات تداول، ولا تُعد
> استشارة مالية.** الهدف تعليمي بحت: تعليم مبادئ إدارة رأس المال وتقليل
> التداول العاطفي.

## المزايا التقنية

- عربية بالكامل، اتجاه RTL افتراضي
- تصميم متجاوب يبدأ من الشاشات الصغيرة (mobile-first)
- تعمل بالكامل من طرف المتصفح (client-side) — بدون خادم أو قاعدة بيانات
- مفتوحة المصدر بالكامل، وقواعد التوزيع مكتوبة بشكل واضح وقابل للتعديل

## البنية (Architecture)

المشروع مقسّم إلى ثلاث طبقات واضحة:

```
src/
  domain/      منطق حساب التوزيع (دوال نقية، بدون واجهة أو حالة)
  config/      قواعد التوزيع والنصوص القابلة للتعديل
  components/  واجهة المستخدم (React)
```

- **Domain**: `src/domain/allocation.ts` يحتوي دالة `calculateAllocation`
  النقية التي تُحوّل رأس المال ونمط المخاطرة إلى أربع فئات، مع اختبارات
  في `allocation.test.ts`.
- **Config**: `src/config/allocationRules.ts` يحدد النسب المئوية لكل نمط،
  و`src/config/copy.ts` يحتوي كل النصوص الظاهرة للمستخدم — مفصولة عن
  المنطق والواجهة لتسهيل التعديل والترجمة مستقبلاً.
- **Components**: عناصر واجهة بسيطة (`CapitalForm`, `AllocationResults`,
  `BucketCard`, `Disclaimer`) تستهلك طبقتي `domain` و`config` دون منطق
  حساب داخلها.

## التشغيل محليًا

يتطلب Node.js 20+.

```bash
npm install
npm run dev       # تشغيل بيئة التطوير
npm run build     # بناء نسخة الإنتاج
npm run test      # تشغيل الاختبارات
npm run lint      # فحص الكود
```
